GSB Frais - Maintenance évolutive

Descriptif du projet

Vous êtes chargés de réaliser les demandes d’évolutions émanant des utilisateurs de l’application.

Compétences mises en oeuvre

  • Répondre aux incidents et aux demandes d’assistance et d’évolution

    • Traiter des demandes concernant les applications

  • Développer la présence en ligne de l’organisation

    • Participer à l’évolution d’un site Web exploitant les données de l’organisation

Outils/Plateformes utilisées

  • Architecture MVC

  • Application web GSBFrais

  • Serveur de base de données MySQL

  • Serveur web Apache2

  • PHP

Déroulement du projet

Ce projet a été réalisé en équipe comptant 7 membres : 3 membres côté SISR, 4 côté SLAM . Certaines tâches ont été réalisées par chaque membre de l’équipe tandis que d’autres ont été dispatchées. A chaque début de séance, nous réalisions une mise au point afin de mettre au clair le travail fait, en cours et à faire.

Objectif

L'Objectif de ce projet était situé dans le contexte suivant : l'entreprise GSB (Galaxy Swiss Bourdin) venait de finir de développer son application web et souhaitais la mettre à disposition de tous. Le but était de faire la bascule des fichiers sur des serveurs physiques en s'assurant que tout fonctionne.

Besoins exprimés

Afin que la mission se déroule sans accros, une liste des besoins à été composée par l'équipe de développement à destination de l'équipe réseau.
Cette dernière comprend :

  • Un compte pour l’accès au serveur sécurisé (SSL/SSH) installé sur la VM pour tous les membres de l’équipe de développement (4 comptes).
  • Un serveur MariaDB version 10.4.24 minimum, avec :
    • Création d’un compte ROOT sécurisé par mot de passe Fort pour la base de données “gsbFrais”
    • Création du compte ‘USR_APPLI_GSBFRAIS’ sur MariaDB avec comme privilèges : SELECT, INSERT, UPDATE, DELETE
  • Serveur Web Apache (2.4)
  • PHP version 8.1 minimum
  • Un serveur FTP sécurisé pour le transfert des fichiers de l’application (exemple : WinSCP)
  • Un certificat HTTPS pour l’application web GSBFrais
  • Création d’un Virtual Host dans Apache pour permettre l’accès à l’application en HTTPS (fichier httpd-vhosts)

Documentation

Tout au long du projet, une documentation à été rempli; Celle-ci à pour but d'expliquer chacune des procédures par lesquelles nous sommes passé pour arriver au résultat final. Entre autres, une personne larguée sans les compétences ni connnaissances peut répliquer le travail effectué juste en suivant les étapes de la documentation.
